## 知识点(概括)
- 1、项目架构(mvc)。个人感觉flutter项目，如果没有好的一个架构，可读性很差，代码越多，越难看。
- 2、flutter多语言实践(Android studio Flutter i18n插件)
- 3、插件使用，通过[pub](https://pub.dartlang.org/packages/) 搜索需要的插件
- 4、Flutter 布局控件
- 5、Flutter 动画
- 6、网络请求
- 7、与原生交互通信(methodCall)
- 8、内部通信(notification)
